# Fix Lint Errors: Unexpected any

## Scope
Two instances of `@typescript-eslint/no-explicit-any` violation found:

1. `components/TickerDirectory.tsx:63` — `catch (err: any)`
2. `app/api/history/[ticker]/route.ts:12` — `catch (err: any)`

## Decision
Replace `any` with `unknown` in both `catch` clauses. Use `instanceof Error` guard to safely access `.message`.

## Steps

### 1. Fix `components/TickerDirectory.tsx`
Replace:
```ts
catch (err: any) {
  console.error("Failed loading tickers:", err);
  setError(err.message);
}
```
With:
```ts
catch (err) {
  console.error("Failed loading tickers:", err);
  if (err instanceof Error) {
    setError(err.message);
  } else {
    setError("Failed loading tickers");
  }
}
```

### 2. Fix `app/api/history/[ticker]/route.ts`
Replace:
```ts
} catch (err: any) {
  return NextResponse.json({ error: err.message }, { status: 500 });
}
```
With:
```ts
} catch (err) {
  const message = err instanceof Error ? err.message : "Internal server error";
  return NextResponse.json({ error: message }, { status: 500 });
}
```

## Validation
Run `npx next lint` and verify zero `@typescript-eslint/no-explicit-any` errors.
